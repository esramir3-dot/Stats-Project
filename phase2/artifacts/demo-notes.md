# DormEats Demo Notes

This artifact describes the scenarios used in the Phase 2 prototype demo.

## Scenario set

1. Exam-week vegan student with no-cook or microwave-friendly constraints
2. Halal student with only a microwave and a few staple ingredients
3. Gluten-free student with almost no groceries available

## What the demo is meant to prove

- The recommendation engine respects dietary restrictions.
- The ranking logic favors meals close to the user’s current ingredients.
- The dorm equipment constraint changes results in a meaningful way.
- A restaurant fallback exists even before live API integration.
