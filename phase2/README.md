# DormEats Phase 2 Prototype

This folder contains the Phase 2 MVP progress deliverables for the DormEats project.

## Contents

- `report.md`: Phase 2 written report
- `demo.py`: offline prototype that recommends dorm-friendly meals and a restaurant fallback
- `evaluate.py`: simple benchmark script for Phase 2 evidence
- `data/`: small structured datasets used by the prototype
- `artifacts/`: generated benchmark and demo outputs

## Quick start

No external packages are required.

```bash
cd phase2
python3 demo.py
python3 evaluate.py
```

## What the current prototype does

- Filters recipes by dietary restriction and available equipment
- Ranks meals using ingredient overlap, time, and cost
- Produces a simple grocery gap list for missing ingredients
- Provides static local restaurant suggestions as a fallback

## Current limitations

- Uses a hand-built mini dataset instead of Yelp, Google Maps, USDA, or Hugging Face data
- Uses rule-based ranking rather than an LLM or learned recommender
- No web interface yet

## Reproducibility note

`artifacts/benchmark.md` is generated by `python3 evaluate.py`.
